# Roadmap — FyT Lab Connect

Este roadmap separa la fase académica (actual) de la futura fase comercial, y organiza objetivos por horizonte temporal.

## Corto plazo (0–3 meses)
- Estabilizar navegación y corregir bugs críticos (ej. CTAs que no redirigen).  
- Añadir pruebas unitarias básicas para componentes críticos (Vitest).  
- Integrar linter y formateador en CI (ESLint, Prettier) — si no está presente, definir reglas.

## Mediano plazo (3–9 meses)
- Automatizar pipeline CI/CD: build, tests, lint, deploy a staging.  
- Mejoras de accesibilidad (WCAG básico).  
- Implementar prerender/SSG completo para las rutas públicas principales.
- Introducir documentación técnica (diagramas, contrato de API si aplica).

## Largo plazo (9–24 meses)
- Internacionalización (i18n) completa.  
- Panel de administración para contenidos (opcional).  
- Modelos de monetización controlada: servicios, cursos, consultoría (validar con stakeholders).
- Escalado infra: CDN, cache, monitoreo y observabilidad.

## Diferenciación de fases
- Fase Académica: enfoque en estabilidad, accesibilidad y reproducibilidad de contenido.  
- Fase Comercial: añadir métricas, autenticación avanzada, panel de facturación/opciones comerciales (sólo si se valida con el owner).
